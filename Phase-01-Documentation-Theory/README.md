# Phase 01 — Documentation and Theory

## Goal

Understand the robot and its software stack before running experiments.

This phase focuses on reading documentation, learning robotics vocabulary, and building a foundation in robot hardware, ROS2, localization, mapping, and SLAM.

## Why This Phase Matters

Before using the MentorPi A1 for SLAM, I need to understand what information the robot can sense, how ROS2 organizes robot software, and why localization and mapping are important robotics problems.

## Deliverables

- `Questions.md` — Phase 1 questions.
- `Answers.md` — structured answers to each question.
- `Learning-Log.md` — notes, discoveries, unclear topics, and next steps.

## Topics Covered

- MentorPi A1 hardware and sensors
- LiDAR
- IMU
- Wheel encoders
- ROS2 nodes, topics, messages, and launch files
- Localization
- Mapping
- SLAM
- Simple explanations for non-technical audiences

## Completion Criteria

Phase 1 is complete when I can clearly explain:

- What sensors are on the robot
- What each sensor measures
- Why LiDAR, IMU, and encoders matter
- What ROS2 is and why robotics companies use it
- The difference between localization, mapping, and SLAM
- How to explain these concepts simply
